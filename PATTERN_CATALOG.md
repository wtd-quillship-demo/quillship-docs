## How to read this catalog

Each entry follows the format:

- **Pattern name** — short, memorable
- **What it looks like** — concrete example or signal
- **Why AI struggles with it** — one or two sentences
- **Fix** — what to do instead
- **Spotted by** — contributor first names (with their permission)

## Anti-patterns reference (preview — to be filled in by contributors)

These are the patterns we expect to encounter in the Quillship demo docs and in real audits:

1. **Wall of prose** — multiple paragraphs without headings or bullets
2. **Untagged code block** — fenced code without a language identifier
3. **Vague reference** — "see the appropriate page" without a link
4. **Undefined acronym** — RT, SSE, ACF used without spelling out
5. **Missing prerequisites** — install instructions assume Node/Python is already there
6. **No `llms.txt`** — site has no machine-readable index
7. **Circular FAQ** — "What is X? See our X page."
8. **Broken anchor** — `[link](#section-that-doesnt-exist)`
9. **Missing TL;DR** — long page with no summary at top
10. **Buried lede** — actual answer is in paragraph 4
